Apa itu DevOps?


DevOps is an approach to bridge the gap between software development and operations

it is a response to the interdependence of software development and IT operations

DevOps is not a technology problem, but a business and cultural

  • stresses collaboration between development and operations
  • aims to deliver software and services quicker

    • Tools can help

  • Cloud computing (AWS, Azure, Google Cloud)
  • Infrastructure as code (Chef, Puppet, Ansible, SaltStack)
  • Build & Test using continuous integration (Jenkins)
  • Containerization (Docker)

    Continuous Integration

    Continuous Integration, in its simplest form, involves a tool that monitors your version control system for changes. Whenever a change is detected, this tool automatically compiles and tests your application. If something goes wrong, the tool immediately notifies the developers so that they can fix the issue immediately.

    Tools: Jenkins, Bamboo (atlassian), TeamCity (jetbrains), CircleCI, ParallelCI, Travis CI, AWS CodePipeLine

    Jenkins

    Open source continuous integration tool written in Java.

    There are a lot of plugins available for Jenkins.

    There are easier to use CI software available, but Jenkins is open source, free and still popular.
